# Project Rules

1. Business rules are forbidden inside widgets.
2. Google API calls are forbidden inside widgets.
3. Shift Parser must not depend on Google Calendar.
4. Calendar Engine must not depend on Google Sheets.
5. Domain models must not import Flutter UI libraries.
6. New features require tests.
7. All synchronization plans require preview.
8. Errors shown to users must include a clear next step.
9. Configuration values must be centralized.
10. Architectural changes require an ADR.
11. No credentials, client secrets, access tokens, or private keys may be committed.
12. Logs must avoid exposing sensitive staff data unless explicitly required.
